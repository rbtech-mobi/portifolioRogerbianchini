Finance Tracker
Jetpack Compose Edition

Este projeto foi desenvolvido como Trabalho Final da Disciplina de Pós-Graduação. O objetivo principal foi implementar uma interface totalmente reativa utilizando a stack moderna do Android, focando em gerenciamento de estado e arquitetura MVVM.

🏗️
Arquitetura Declarativa
Compose · ViewModel · State Hoisting
1
Unidirectional Data Flow (UDF)
O estado flui para baixo e os eventos fluem para cima, garantindo que a UI seja uma função direta do estado do ViewModel.
2
Interoperabilidade XML
Utilizei AndroidView para integrar componentes legados dentro do ecossistema Compose, demonstrando flexibilidade técnica para migrações de sistemas complexos.
🎨
Design System & Colors
Material 3 · Custom Theme

A escolha das cores seguiu uma paleta Teal & Dark Grey:

  • ● Teal (#10b981): Utilizado para sinalizar sucesso, liquidez e fluxos financeiros positivos.
  • ● Dark Background: Reduz a fadiga visual e destaca os dados numéricos das transações.
  • ● Typography: IBM Plex Mono para valores, reforçando o aspecto técnico/contábil.
Kotlin · MainActivity.kt 
@Composable
fun MainApp(viewModel: TransactionViewModel = viewModel()) {
    var screenState by remember { mutableStateOf<Screen>(Screen.List) }
    
    when (screenState) {
        is Screen.List -> TransactionListScreen(onNew = { screenState = Screen.Form })
        is Screen.Form -> TransactionFormScreen(onBack = { screenState = Screen.List })
    }
}
Compatibilidade: O projeto utiliza a BOM (Bill of Materials) 2024.09.00, garantindo compatibilidade entre as bibliotecas de UI e Lifecycle, com suporte nativo desde o Android 7.0 (API 24).
