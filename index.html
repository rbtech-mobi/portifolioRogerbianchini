Sales Engineering / AI
Outreach Calibrator
LLM-powered B2B prospecting engine that eliminates generic outreach
The Business Problem:

Sales Development Representatives (SDRs) face a brutal trade-off: spend hours manually researching a single prospect, or send generic, low-converting automated spam. The lack of context kills the pipeline.

The Architecture:

A frontend interface that acts as an ingestion layer for monday.com CRM webhooks. It vectorizes structured demographic data (role, company) alongside unstructured Natural Language Processing (NLP) inputs—such as a prospect's recent LinkedIn comments or company news. It then queries a connected Large Language Model (LLM) to map the prospect's real-time pain points directly to the seller's specific value proposition.

Business Impact: Demonstrates the perfect intersection of sales strategy and software engineering. It automates the SDR research workflow without losing human context, drastically increasing reply rates by ensuring every approach is highly calibrated to current events.

Full-Stack / Architecture
Multi-Tenant SaaS Engine
One codebase securely serving isolated independent businesses
The Business Problem:

Scaling a B2B platform requires absolute trust. If one client's data leaks into another's dashboard, the business dies. Furthermore, handling concurrency in e-commerce (two people buying the last item at the same time) often leads to cancelled orders and damaged brand reputation.

The Architecture:

A monolithic Node.js backend using a disciplined 4-layer routing pipeline (Express 5 + Zod). Identity is verified via Firebase, but authorization is handled entirely in MySQL. The system enforces strict tenant isolation: cross-tenant lookups return a 404, never a 403, preventing resource leakage. Financials are handled via an append-only event ledger, processing all money as integer cents to avoid floating-point binary errors.

Business Impact: Proves the ability to make hard engineering decisions that protect revenue. Utilizing server-priced idempotency guarantees no double-charging, and the append-only ledger provides enterprise-grade auditability.

IoT / Edge Computing
Desktop Buddy BTTF
Real-time MQTT hardware interface for localized workflow alerts
The Business Problem:

Critical system alerts and workflow approvals get lost in overflowing email inboxes and noisy Slack channels. When response time is critical, software-only notifications fail to capture operator attention.

The Architecture:

An interactive, physical desktop companion running on bare-metal C++ (ESP32-S3). It utilizes enterprise MQTT patterns (QoS 1, HiveMQ TLS) to receive real-time notifications from an n8n orchestration server. It features a FreeRTOS state machine, touch-screen action responses, and a robust dual-network failover system to ensure it never loses connection to the broker.

Business Impact: Brings critical operational data to the physical edge with zero latency. It proves deep capability in hardware-software integration, network reliability, and securing IoT endpoints via certificate validation.
ESP32-S3 C++ / FreeRTOS MQTT (TLS) LovyanGFX n8n Orchestration
